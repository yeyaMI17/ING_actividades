## **Tema: ¿Qué es JSON y XML?**  
Contar con protocolos específicos para el **intercambio de datos** es fundamental para que se pueda producir el proceso con todas las garantías de éxito, sea cual sea el lenguaje de programación en el que se base el software empleado o el sistema operativo que use cada ordenador para funcionar.  
**JSON** y **XML** son tecnologías web populares.  
A continuación, se dará información detallada de cada una ya que es indispensable conocerlas para elegir el más adecuado en cada situación.

### **INICIOS**  
A principios de los años 90 surgió el problema de que las máquinas pudieran entenderse entre sí. Utilizaban diferentes sistemas operativos y sus programas estaban escritos en diferentes lenguajes de programación.  
Una de las soluciones fue crear el estándar **XML** y más tarde **JSON**, para a través de ellos poder intercambiar información entre las máquinas independientemente del sistema operativo o del lenguaje de programación con el que estuviera hecho el software.  
  
## **XML**  
**XML** es un lenguaje extensible. Se basa en un **metalenguaje** que ayuda a definir lenguajes desarrollados por el World Wide Web Consortium usado para almacenar contenidos, datos o referencias en forma legible.  

Es una clasificación del **SGML**, este lenguaje ayuda en la clasificación y el etiquetado de títulos. Esto quiere decir que el este lenguaje no es un idioma en sí mismo, es más bien un sistema que permite precisar lenguajes basados en las necesidades. El **XHTML**, el **SVG** y **MathML** son varios de los lenguajes que domina y logra definir.  

![Mi Imágen](/archivos/individual/act5/xml.jpg)
### **Características**
**1.** Separa los datos de HTML.  
De esta forma, logra reunirse en el uso de HTML y CSS para una visión y diseño completo, tomando en cuenta las modificaciones en los antecedentes inferiores no soliciten cambios en el HTML.  
**2.** Simplifica el intercambio de datos.    
os antecedentes XML se acopian en forma de texto plano. Esto suministra una forma independiente de software y hardware para almacenar datos.  
**3.** Simplifica el transporte de datos.  
La transferencia de informes XML domina extraordinariamente esta complejidad, ya que los registros logran ser leídos por variadas aplicaciones indebidas.  
**4.** Simplifica el cambio de plataforma.  
Suministra la actualización a desconocidos sistemas operativos, nuevas aplicaciones o desconocidos navegadores, sin perder datos.  
**5.** XML aumenta la disponibilidad de datos.  
Con este lenguaje, sus registros logran estar disponibles para todo tipo de aparatos de lectura como monitores de mano, máquinas de voz, arranques de noticias, entre otros y hacer que estén más disponibles para personas con una visión compleja o con otras discapacidades.  
**6.** XML logra ser utilizado para crear nuevos idiomas de internet. Por ejemplo: 

* XHTML 
* WSDL 
* WAP y WML
* RSS (utilizado en noticias)
* RDF y OWL
* SMIL. 
### **Usos**  
* El uso principal de XML es estructurar datos, recibirlos y/o enviarlos, pero también podemos guardar datos en nuestros documentos para que sean tratados luego con X lenguaje.  
* Se utiliza para varios servicios web. Tanto es así que hay sistemas destinados a ayudar en la definición de lenguajes basados ​​en XML, así como APIs que ayudan en el procesamiento de datos XML (que no deben confundirse con HTML).

## **JSON** 
**JSON**, cuyo nombre corresponde a las siglas **JavaScript Object Notation** o Notación de Objetos de **JavaScript**, es un formato ligero de intercambio de datos, que resulta sencillo de leer y escribir para los programadores y simple de interpretar y generar para las máquinas.  
JSON es un formato de texto completamente independiente de lenguaje, pero utiliza convenciones que son ampliamente conocidos por los programadores, entre ellos:

* C  
* C++  
* C#  
* Java  
* JavaScript  
* Perl  
* Python  
* Entre otros  

Dichas propiedades hacen de JSON un formato de intercambio de datos ideal para usar con API REST o AJAX. A menudo se usa en lugar de XML, debido a su estructura ligera y compacta.  

JSON se caracteriza por reducir el tamaño de los archivos y el volumen de datos que es necesario transmitir. Por ello fue adquiriendo popularidad hasta convertirse en un estándar. Esto no significa que XML haya dejado de utilizarse, en la actualidad ambos se emplean para el intercambio de datos.  

### **Características**  
* JSON es solo un formato de datos.
* Requiere usar comillas dobles para las cadenas y los nombres de propiedades. Las comillas simples no son válidas.
* Una coma o dos puntos mal ubicados pueden producir que un archivo JSON no funcione. 
* Puede tomar la forma de cualquier tipo de datos que sea válido para ser incluido en un JSON, no solo arreglos u objetos. Así, por ejemplo, una cadena o un número único podrían ser objetos JSON válidos.
* A diferencia del código JavaScript, en el que las propiedades del objeto pueden no estar entre comillas, en JSON solo las cadenas entre comillas pueden ser utilizadas como propiedades.  
### **Usos** 
* Si está escribiendo software que se comunica con un navegador nativo o una aplicación móvil, debe utilizar JSON como formato de datos.  
* El uso de JSON está creciendo rápidamente en toda la industria, debido a que es una tecnología útil para los desarrollos. 
Recientemente Twitter migró de XML a JSON. Google Web Toolkit también trabaja con este formato.
